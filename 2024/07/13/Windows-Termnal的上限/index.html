OVERVIEW

WindowsTerminal

简称wt,一款由微软开发的windows终端，相对于windows上的其他我认为它有着更好的操作界面以及更直白的选项卡，体现于目前主流的终端模拟器所没有的用选项卡改配置，以及流畅的gui动画，还有高（好吧一点都不高）可定制性.
在Win11上它的表现尤其突出，
配合窗口圆角以及专注模式可以实现无边框的沉浸式圆角窗口，同时高版本的powershell所提供的现代化终端体验丰富了其华丽外衣下的空洞内在，
使用Startship之后更是锦上添花般地给予了我极好的终端体验，
常言道，美化是第一生产力，
你的效率取决于你有多喜欢这件事，那么有一个漂亮的终端就能极大提升你写程序的效率。

Nvchad

一个现代化的nvim配置，有优美的UI和完备的写码功能
官网
唯一不好的地方就是第一次接触这个配置的哥们可能不太能明白该怎么修改配置，
其实主要就是在lua文件夹下的chadrc.lua(用来改ui)文件和主文件夹下的init.lua文件
我将自己的这两个文件贴出供各位参考

init.lua

vim.g.base46_cache = vim.fn.stdpath "data" .. "/nvchad/base46/"
vim.g.mapleader = " "
--require('lua.custom.init')
-- bootstrap lazy and all plugins
local lazypath = vim.fn.stdpath "data" .. "/lazy/lazy.nvim"
if not vim.loop.fs_stat(lazypath) then
  local repo = "https://github.com/folke/lazy.nvim.git"
  vim.fn.system { "git", "clone", "--filter=blob:none", repo, "--branch=stable", lazypath }
end

vim.opt.rtp:prepend(lazypath)
local lazy_config = require "configs.lazy"

-- load plugins
require("lazy").setup({
  {
    "NvChad/NvChad",
    lazy = false,
    branch = "v2.5",
    import = "nvchad.plugins",
    config = function()
      require "options"
    end,
  },

  { import = "plugins" },
}, lazy_config)

-- load theme
dofile(vim.g.base46_cache .. "defaults")
dofile(vim.g.base46_cache .. "statusline")

require "nvchad.autocmds"
vim.schedule(function()
  require "mappings"
end)
